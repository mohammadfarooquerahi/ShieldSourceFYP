Q1 What AI/ML technique did you use and why did you choose it?

I used a Random Forest Classifier — an ensemble learning method that builds 200 independent decision trees and takes a majority vote.

I chose Random Forest over alternatives like SVM or Neural Networks for three reasons:

① It works well with small tabular datasets — my feature vector is only 7 numbers
② It provides probability scores via predict_proba() — this gives me the confidence percentage shown to the expert
③ It is resistant to overfitting through bootstrap aggregation — each tree trains on a random subset of data

Q2 What dataset did you use to train the model?

I used synthetic training data generated programmatically in train_model.py. I generated 700 labeled samples — 140 per class — with realistic statistical ranges based on real attack documentation.

Why synthetic? Real attack log datasets like CICIDS2017 are gigabytes in size, many require institutional access, and real logs contain PII — real IP addresses and real usernames. Using them would raise ethical and legal concerns. Synthetic data is a standard academic approach for demonstrating ML pipelines while keeping the dataset controllable and explainable.

Q3 What accuracy did your model achieve?

The model achieved 100% accuracy on the test set — 140 test samples with perfect precision, recall, and F1-score for all 5 classes.

This is expected with synthetic data because the class boundaries are well-defined. For example, a log with 50 SQL keywords will never be confused with a normal log that has 0-2. In a real-world deployment, accuracy would typically be 85-95% due to noise and mixed patterns in real logs. For my FYP, synthetic data proves the full pipeline works correctly end-to-end.

Q4 What are the 7 features? Why these specific ones?

The 7 features were chosen based on known cybersecurity attack signatures:

total_lines
Total line count in the log file
Baseline volume indicator
error_lines
Lines containing ERROR, FATAL, CRITICAL
Exploitation attempts generate errors
sql_keywords
Lines with SELECT, UNION, DROP, INSERT
→ Detects SQL Injection attacks
auth_failures
Lines with "failed login", "unauthorized"
→ Detects Brute Force attacks
unique_ips
Count of distinct IPv4 addresses
→ Many IPs = DDoS botnet
high_freq_ip_lines
Lines from IPs appearing 10+ times
→ Single-source DDoS flood
path_traversal
Lines with ../ or ..\ sequences in URL requests
→ Detects Directory Traversal attacks

Q6 How does the ML service connect to the main application?

The system uses a microservice architecture with 3 separate services running simultaneously:

① React Frontend (port 3000) — what the user sees in the browser
② Node.js API (port 5000) — handles authentication, database, business logic
③ Python Flask ML Service (port 8000) — handles AI classification only

When a user uploads a log file, Node.js saves it to disk, then makes an HTTP POST request to localhost:8000/analyze with the file path. Flask reads the file, extracts 7 features, runs the Random Forest model, and returns a JSON result. Node.js saves the result to the ml_predictions table in MySQL.

Q7 What is SHA-256 and why do you use it?

SHA-256 is a cryptographic hash function that produces a unique 64-character fingerprint for any file. Even changing one character completely changes the hash.

In cybersecurity forensics, file integrity is critical. If a log file is submitted as evidence, we must prove it hasn't been modified. By storing the SHA-256 hash at upload time, anyone can later re-hash the file and confirm it matches — proving the evidence is authentic and unmodified. This is the same technique used by law enforcement in digital forensics.

Key Technical Terms
Know these definitions confidently
Random Forest
200 decision trees that each vote on the answer — majority vote wins. More trees = more accurate and stable.
Feature Engineering
Converting raw log text into 7 numbers that the ML model understands and can learn from.
Synthetic Data
Programmatically generated training data with realistic patterns. Used when real data is scarce or contains PII.
Microservice
Small, separate service (Python Flask on port 8000) doing ONE specific job — ML prediction only.
SHA-256
Cryptographic hash — 64-character fingerprint proving a file hasn't been modified. Standard in digital forensics.
predict_proba()
sklearn function returning probability for each class. The max value becomes the confidence score (e.g. 97%).
Confidence Score
How certain the model is in its prediction. 97% = very confident. Stored in ml_predictions table.
JWT
JSON Web Token — encrypted login token stored in browser. Sent with every API request to prove identity.
ENUM (MySQL)
Column type accepting only specific values: open / in_progress / resolved. Prevents invalid data.
bcrypt
Password hashing algorithm. Passwords are never stored as plain text — only as a 60-char bcrypt hash.
